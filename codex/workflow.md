# Workflow Details

## Task Management Steps

1. Write plan to `tasks/todo.md` with checkable items.
2. Check in with user before starting implementation.
3. Mark items complete as you go; explain changes at each step.
4. Add review section to `tasks/todo.md` when done.
5. Capture lessons in `tasks/lessons.md` after any correction.

## Collaboration

- Escalate when requirements are ambiguous, security-sensitive, or would change UX/API contracts.
- Scoring mindset: wrong code = -4 pts, successful change = +1 pt, honest uncertainty = 0 pts.
- Value correctness over speed.

## Quality Check

- For non-trivial changes, pause and ask: "Is there a more elegant way?"
- Skip this for simple, obvious fixes — don't over-engineer.
- Diff behavior between main and your changes when relevant.
